Java Generic, 제네릭 타입

정의 및 기능에 대한 간단한 설명 중심으로

Posted on February 24, 2021

제네릭이란?

하나 이상의 타입 매개변수(type parameter)를 선언하고 있는 클래스나 인터페이스를 제네릭 클래스, 또는 제네릭 인터페이스라고 하며 이를 합쳐 제네릭 타입이라고 한다.

여기서 타입 매개변수(type parameter)는 <>안에 있는 타입을 말한다.

사용 목적

외부에서 형을 결정할 수 있다는 점과 오류가 발생했을 때 컴파일 타임에 알 수 있기때문이다.

제네릭없이 Object를 이용하여 외부에서 내부 멤버 변수의 형을 결정할 수 있다. 하지만 오류가 발생했을 때 런타임 시간에 체크해야하기 때문에 디버깅을 통해 체크해야하며, 상당한 시간이 걸린다. 제네릭을 사용하면 컴파일할 때 알 수 있기 때문에 바로 해결할 수 있는 장점이 있다.

형식 타입 매개변수

제네릭 타입을 선언할 때 사용하는 것으로, 명명규칙을 다음과 같다.

  • E : Element -> Collection 정의할 때
  • K : Key -> Collection 중에 Key가 필요한 경우
  • N : Number
  • T : Type -> 일반적인 제네릭 정의할 때 사용
  • V : Value

와일드 카드 타입

와일드 카드 타입을 설명하기 앞서 공변과 불공변을 설명한다.

공변은 변수를 공유할 수 있다는 의미이다. 대개 부모 자식 클래스 관계에서 부모 변수는 자식 객체를 받아 들일 수 있다. 대표적은 Object 변수가 이에 해당된다.

반대로 공유할 수 없는 변수를 불공변이라고 한다. 아무 관계 없는 클래스 관계에서 변수에 다른 객체를 할당 할 수 없다.

제네릭 타입은 대표적인 불공변이다.

예를 들면 List<Integer>와 List<String>의 객체는 불공변이기 때문에 서로의 변수에 할당 할 수 없다. 이를 위한 메커니즘이 바로 와일드 카드 타입이다.

ex) List<?>=new List<Integer>();나 List<?>=new List<String>(); 이런식으로 사용 가능하다.

따라서 와일드 카드 타입은 변수 및 매개 변수 선언시 주로 사용된다.

바운디드

타입 매개변수의 범위를 한정 짓기 위해 형식 타입 매개변수나 와일드 카드에 사용되며 extends와 super를 통해 타입 매개변수의 범위를 한정 지을 수 있다.

바운디드 와일드 카드매개변수 예시

  • public void a(List<? extends A> b){ ... } -> A를 포함한 자식들까지로 한정
  • public void a(List<? super A> b){ ... } -> A를 포함한 부모들까지로 한정

바운디드 형식 타입 매개변수

  • public class A<T extends B>{ ... } -> B를 포함한 자식들까지로 한정
  • public class A<T extends B>{ ... } -> B를 포함한 부모들까지로 한정

제네릭을 사용할 수 없는 경우

new와 형식적 매개변수를 이용해 객체를 생성할 수 없다. ex) T[] a = new T[5](X);
static 변수에도 제네릭을 사용할 수 없다.

이는 제네릭은 런타임 시간에 타입이 결정되는 데 위의 두가지 경우는 컴파일 타임에 결정되기 때문이다. 또한 Collection의 객체들은 컴파일 시간이 지나면 타입 매개변수가 없어져 있다. 타입이 이미 결정됬기 때문이다.

메서드와 제네릭

제네릭 메서드는 static이든 아니든 사용가능하다.
메서드는 변수처럼 형을 정해서 메모리에 할당하는 것이 아닌 틀만 메모리에 할당하기 때문이다.

기본적으로 return type 앞에 <>안에 형식 타입 매개변수를 정의한다. 그리고 클래스 역시 제네릭인데 내부에 제네릭 메서드를 정의하게 될 때 둘은 전혀 다른 것으로 인식된다.
