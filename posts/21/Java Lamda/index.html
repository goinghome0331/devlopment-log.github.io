Java Lamda, 람다함수

개념과 사용법 중심으로

Posted on March 2, 2021

람다 함수란?

람다 함수는 수학 분야에서 람다 대수에 근간을 두고 있으며, 자세한 의미는 매우 어려우므로 넘어가도록 한다. 컴퓨터 분야에서는 익명 함수를 지칭한다고 생각하면 된다.
여기서 익명함수는 말 그대로 이름이 없는 함수이며 1급 객체이기도 한다.

여기서 잠시 1급 객체의 특징을 살펴보자.

  • 변수나 데이터에 할당할 수 있어야한다.
  • 객체의 인자로 넘길 수 있어야 한다.
  • 객체의 리턴값으로 리턴할 수 있어야 한다.

함수형 언어에서 위와 같은 특성이 있기때문에 익명함수는 함수형 언어에서 주로 가능하다.

하지만 자바는 함수형 언어가 아니라 객체 지향 언어이다. 그렇다면 자바의 람다함수는 어떻게 가능할까? 다음과 같은 제한 조건을 통해 구현할 수 있다.

제한 조건

interface를 통해 구현해야 하며, 그 interface는 반드시 하나의 추상 메서드만을 가져야 한다. 그리고 이를 강제화 하려면 interface 위에 FunctionalInterface라는 어노테이션을 사용한다.

사용법

기본적으로 (매개변수) -> {함수몸체} 의 형태로 사용한다.

각 부분 생략 가능한 경우가 있다.

  • 소괄호 생략 : 단일 매개변수 일 때
  • 자료형 생략 가능 : 매개변수의 타입을 추론할 수 있는 경우, 변수명만 사용 가능
  • 중괄호 생략 : 본문이 한줄일 때
  • return 생략 : 본문 한 줄에 return 문만 존재할 때
  • 세미콜론 생략 : 중괄호나 return이 생략된 경우

생략 불가능 한경우

  • 매개 변수가 없거나, 두 개 이상인 경우
  • 중괄호 : 두 개 이상의 실행문이거나 한 줄의 return 문이 존재한 경우

사용 예

Collection의 stream api에서 사용되는 것이 대표적인 예이다.

Predicate와 Consumer가 대표적인 람다 함수용 인터페이스이다.
