Java Static

사용되는 위치 중심으로...

Posted on March 19, 2021

static이란?

자바의 정해진 키워드 중 하나이며, 대개 프로그램이 실행되는 동안 유지하거나 또는 전역적으로 접근하기 위한 용도로 사용된다.

JVM 메모리 상에는 Method 영역에 위치하게 된다.

사용되는 곳

사용되는 곳은 총 5가지이다.

  • 변수
  • 클래스
  • 메소드
  • import
  • 블록

변수

메모리에 고정적으로 한번만 할당되어, 프로그램이 종료될 때 해제되는 변수

그렇기 때문에 해당 클래스로 생성된 모든 객체가 이 변수를 사용할 수 있다.

또한 객체를 생성하지 않고 클래스 명으로 접근가능하다. 물론 접근지정자에 따라 다르다.

클래스

클래스 명 앞에 static을 붙이면 static 클래스이고 아니면 non static 클래스이다.
하지만 우리가 한 자바 파일에 만드는 클래스는 아무것도 적히지 않았기 때문에 Non static인 거 같지만 static 클래스이다. 또한 static을 붙일 수 없다.

한 클래스 안에 내부 클래스를 정의할 수 있는데 여기에는 static과 non static이 둘 다 존재한다.

특징은 내부 static 클래스는 일반적으로 선언하는 클래스와 같다.
하지만 non static 클래스는 외부 클래스가 객체로 생성된 후 이를 통해 생성할 수 있다. 또는 외부 클래스의 메서드를 통해 생성될 수 있다.

메서드

클래스 함수라고 불리며 변수와 같이 객체 생성 없이 사용가능하다.

static 메서드는 내부 클래스의 인스턴스 변수를 접근 할 수 없다.(이유는 간단히 설명하자면 변수의 생명주기와 안맞기 때문에)

대개 Math와 같은 유틸리티 클래스에서 주로 구현된다. 대게 전역적으로 사용되고 내부 멤버 변수와 함께 사용되기 보다는 그 함수만으로 기능을 제공하기 때문이다.
따라서 유틸리티 클래스 상속될 필요가 없기 때문에 final을 붙인다.

또한 싱글톤 패턴에서 사용된다. 유일하게 생성될 수 있는 객체에 전역적으로 접근해야하기 때문이다.

import

해당 클래스의 static 메서드를 클래스 명을 없애고 싶을 때, 주로 사용한다.

static import에는 일반적으로 *를 사용하지 않는 것을 권장한다.

static import는 Eclipse에서 자동완성이 안된다.

주로 JUnit Test에서 사용된다.

블록

static{ } 이런식으로 사용되며, 클래스 내부 초기화 작업이 이뤄진다.

프로그램의 시작점인 main보다 앞서 실행된다.
